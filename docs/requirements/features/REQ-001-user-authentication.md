## 功能概述

### 功能描述

用户认证是 OSRM 系统的安全入口模块，提供用户身份验证、会话管理和安全控制功能。所有其他功能都依赖于用户认证，是系统的基础前置功能。

### 业务价值

- 保护系统资源，防止未授权访问
- 识别用户身份，支持权限控制
- 支持审计追踪和操作日志

### 用户故事

- 作为系统用户，我想要通过用户名密码登录系统，以便访问授权的功能
- 作为系统用户，我想要保持登录状态，以便在一段时间内无需重复登录
- 作为系统用户，我想要安全退出系统，以便保护我的账号安全
- 作为系统管理员，我想要控制用户并发会话数，以便防止账号共享

## 业务规则

| 规则编号 | 规则描述 | 优先级 |
|----------|----------|--------|
| BR-001-001 | 用户密码必须经过 bcrypt 加密存储，cost factor = 12 | 高 |
| BR-001-002 | JWT Access Token 有效期为 2 小时，Refresh Token 有效期为 7 天 | 高 |
| BR-001-003 | 连续 5 次登录失败后，账号锁定 30 分钟 | 高 |
| BR-001-004 | 密码必须包含至少 8 个字符，包含字母和数字 | 中 |
| BR-001-005 | 密码 90 天强制更换，不能与前 3 次重复 | 中 |
| BR-001-006 | 单用户最大并发会话数 3 个 | 中 |
| BR-001-007 | 30 分钟无操作自动登出 | 中 |

## 功能需求

### REQ-001-001: 用户登录

**描述**: 用户通过用户名和密码登录系统，成功后返回双 Token

**前置条件**:
- 用户已注册且账号状态正常
- 账号未被锁定
- 未达到最大并发会话数

**后置条件**:
- 生成 JWT Access Token 和 Refresh Token
- 记录登录日志
- 更新最后登录时间

**界面原型**: 登录页面

#### 登录流程

```
用户输入用户名密码
    ↓
验证账号状态（是否锁定/禁用）
    ↓
验证密码
    ↓
检查并发会话数
    ↓
生成双 Token
    ↓
记录登录成功日志
    ↓
返回 Token 和用户信息
```

#### 异常处理

| 异常场景 | 错误提示 | 处理 |
|----------|----------|------|
| 用户名不存在 | "用户名或密码错误" | 记录失败日志，不提示用户名不存在 |
| 密码错误 | "用户名或密码错误" | 失败次数+1，达到5次锁定账号 |
| 账号已锁定 | "账号已锁定，请 30 分钟后重试" | 拒绝登录，提示剩余锁定时间 |
| 账号已禁用 | "账号已被禁用，请联系管理员" | 拒绝登录 |
| 并发会话超限 | "登录设备数已达上限，请先退出其他设备" | 提供踢出其他设备选项 |

---

### REQ-001-002: Token 刷新

**描述**: 使用有效的 Refresh Token 换取新的 Access Token

**前置条件**:
- Refresh Token 未过期
- Refresh Token 未被吊销
- 用户账号状态正常

**后置条件**:
- 生成新的 Access Token
- Refresh Token 剩余有效期保持不变

**业务流程**:
1. 前端检测到 Access Token 即将过期（过期前5分钟）
2. 调用刷新接口，传入 Refresh Token
3. 后端验证 Refresh Token 有效性
4. 生成新的 Access Token 返回
5. 前端更新存储的 Access Token

---

### REQ-001-003: 用户登出

**描述**: 用户主动退出系统，吊销当前会话

**前置条件**:
- 用户已登录

**后置条件**:
- Access Token 加入黑名单（TTL = Token 剩余有效期）
- Refresh Token 从 Redis 中删除
- 清除客户端 Token 存储
- 记录登出日志

---

### REQ-001-004: 登录状态检查

**描述**: 验证 Access Token 有效性，返回当前用户信息

**场景**:
- 页面刷新后恢复登录状态
- 路由切换时验证权限

**返回信息**:
- 用户基本信息（ID、用户名、显示名）
- 角色列表
- 权限列表

## 验收标准

### AC-001: 正常登录

**Given** 用户已注册且账号状态正常
**When** 用户使用正确的用户名和密码登录
**Then** 系统返回 JWT Access Token 和 Refresh Token
**And** Access Token 有效期为 2 小时，包含用户角色信息
**And** Refresh Token 有效期为 7 天

### AC-002: 登录失败处理

**Given** 用户已注册
**When** 用户使用错误的密码登录
**Then** 系统返回错误码 40101001，提示"用户名或密码错误"
**And** 记录登录失败日志，失败次数+1

### AC-003: 账号锁定

**Given** 用户连续 5 次登录失败
**When** 用户再次尝试登录
**Then** 系统返回错误码 42901001，提示"账号已锁定，请 30 分钟后重试"
**And** 锁定期间任何登录尝试都被拒绝

### AC-004: Token 刷新

**Given** 用户已登录且 Refresh Token 有效
**When** 用户请求刷新 Token
**Then** 系统返回新的 Access Token
**And** Refresh Token 剩余有效期保持不变
**And** 原 Access Token 加入黑名单

### AC-005: 用户登出

**Given** 用户已登录
**When** 用户调用登出接口
**Then** 当前 Access Token 被加入黑名单
**And** Refresh Token 被删除
**And** 使用原 Token 访问受保护资源返回 401

### AC-006: 并发会话限制

**Given** 用户已在 3 个设备登录
**When** 用户在第 4 个设备尝试登录
**Then** 系统提示"登录设备数已达上限"
**And** 提供踢出其他设备的选项

## 非功能需求

### 性能要求
- 登录接口响应时间 < 500ms
- Token 刷新接口响应时间 < 200ms
- 支持 1000 并发登录

### 安全要求
- 密码必须使用 bcrypt 加密，cost factor = 12
- Token 传输必须使用 HTTPS
- JWT 必须包含签名，使用 HS256 算法
- Token 黑名单使用 Redis 存储

### 审计要求
- 所有登录尝试记录日志（成功/失败）
- 日志内容包括：用户名、IP 地址、时间、结果、失败原因
- 日志保留 1 年
