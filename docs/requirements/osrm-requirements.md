## 1. 项目背景与目标

### 1.1 背景
企业内部开源软件管理存在以下痛点：
- 开源软件分散存储，缺乏统一管理
- 开发人员重复下载，无法追踪使用情况
- 安全合规难以把控，缺乏审计手段
- 不知道哪些业务系统使用了哪些开源组件

### 1.2 目标
构建统一的开源软件管理平台，实现：
- 统一纳管 Harbor、Nexus、NAS 等存储后端
- 开源软件全生命周期管理（录入-审核-发布-订购-使用）
- 使用情况的追踪与统计
- 安全合规的基础支撑（后续对接第三方扫描）

## 2. 角色与权限

| 角色 | 权限范围 | 主要职责 |
|-----|---------|---------|
| **访客**（未登录） | 只读：查看开源软件介绍、分类、基本信息 | 浏览了解平台资源 |

| **系统管理员** | 全部功能（配置管理、用户管理、审批发布、用量监控） | 平台配置、纳管存储后端、审批发布 |

## 4. 功能模块

### 4.0 用户认证（基础功能）

**需求编号**: REQ-001

用户认证是所有功能的前置依赖，提供安全的身份验证和会话管理。

#### 4.0.1 登录方式

| 方式 | 优先级 | 说明 |
|------|--------|------|
| 用户名/密码登录 | P0 | 基础登录方式 |
| LDAP/AD 集成 | P2 | 对接企业统一认证 |
| 单点登录 SSO | P3 | OAuth2/SAML 集成 |

#### 4.0.2 密码策略（REQ-001-001）

| 策略项 | 规则 | 错误提示 |
|--------|------|----------|
| 最小长度 | 8位 | "密码长度至少8位" |
| 复杂度 | 必须包含字母和数字 | "密码必须包含字母和数字" |
| 密码过期 | 90天强制更换 | 过期前7天提醒 |
| 历史密码 | 不能与前3次重复 | "不能使用近期使用过的密码" |
| 锁定策略 | 5次失败后锁定30分钟 | "账号已锁定，请30分钟后重试" |

#### 4.0.3 会话管理（REQ-001-002）

| 配置项 | 规则 |
|--------|------|
| Access Token 有效期 | 2小时 |
| Refresh Token 有效期 | 7天 |
| 并发会话限制 | 单用户最大3个会话 |
| 超时自动登出 | 30分钟无操作自动登出 |
| Token 续期 | 有效期内活跃自动续期 |

#### 4.0.4 登出与吊销（REQ-001-003）

- 用户主动登出：立即吊销当前 Token
- 异地登录：可选是否踢掉其他会话
- 管理员强制登出：可强制用户下线

### 4.1 存储后端管理（系统管理员）

**需求编号**: REQ-100 ~ REQ-199

**需求编号**: REQ-100 ~ REQ-199

**功能描述**：纳管 Harbor、Nexus、NAS/本地存储等存储后端

**配置项**：

#### Harbor 仓库配置
- 仓库名称（显示名）
- Harbor 地址（IP/域名 + 端口）
- 访问协议（HTTP/HTTPS）
- 认证信息（用户名/密码 或 Token）
- 项目/组织名称（默认存储位置）
- 仓库状态（在线/离线）
- 连接测试按钮

#### Nexus 仓库配置
- 仓库名称（显示名）
- Nexus 地址（IP/域名 + 端口）
- 访问协议（HTTP/HTTPS）
- 认证信息（用户名/密码）
- 仓库类型映射：
  - Maven hosted 仓库名称
  - NPM hosted 仓库名称
  - PyPI hosted 仓库名称
  - Raw hosted 仓库名称
- 仓库状态（在线/离线）
- 连接测试按钮

#### NAS/本地存储配置
- 存储名称
- 存储类型：NAS挂载、本地磁盘
- 访问路径（挂载点、本地路径）
- 存储容量限制
- 当前使用量

#### 附件存储配置
- 存储路径（本地路径，如 `/data/attachments`）
- 单文件大小限制：100MB
- 总容量限制

**功能操作**：
- 新增存储后端
- 编辑配置
- 删除存储（需确认无关联制品）
- 连接测试（实时验证配置是否正确）
- 查看存储用量统计

**存储后端健康检查**（REQ-102）：
- 自动健康检测：每5分钟检测一次存储后端连通性
- 健康状态：在线/离线/异常
- 异常告警：离线时发送通知（邮件/站内信）
- 离线处理：离线存储后端上的软件自动标记为"暂不可用"

### 4.3 开源软件管理（软件管理员）

**需求编号**: REQ-300 ~ REQ-399

#### 4.3.1 软件录入

**需求编号**: REQ-300

**基本信息**：
- 软件名称（必填）
- 显示名称（用于展示）
- 软件类型（自动映射存储后端）：
  - Docker 镜像 → 存储到 Harbor
  - Helm Chart → 存储到 Harbor
  - Maven 组件（JAR/WAR/POM） → 存储到 Nexus
  - NPM 包 → 存储到 Nexus
  - PyPI 包 → 存储到 Nexus
  - 通用文件 → 存储到 NAS

**存储后端自动映射规则**：
| 软件类型 | 存储目标 | 说明 |
|---------|---------|-----|
| Docker 镜像 | Harbor | 推送到 Harbor 的 project |
| Helm Chart | Harbor | Harbor 2.5+ 支持 Helm 仓库 |
| Maven | Nexus | 推送到 maven-hosted 仓库 |

**版本号规范**（REQ-301）：
- 遵循语义化版本控制（SemVer）：`主版本.次版本.修订号[-预发布标识]`
- 有效示例：`1.0.0`, `2.1.3`, `1.0.0-beta`, `2.0.0-rc.1`
- 系统自动校验版本号格式
- 同一软件禁止重复版本号录入
- 版本比较支持：系统能正确比较 `1.0.0` < `1.0.1` < `1.1.0` < `2.0.0`
| NPM | Nexus | 推送到 npm-hosted 仓库 |
| PyPI | Nexus | 推送到 pypi-hosted 仓库 |
| 通用文件 | NAS | 存储到 NAS 挂载目录 |
- 版本号（必填）
- 描述说明
- 标签/分类（多选）：操作系统、数据库、中间件、开发工具等

**制品上传**：
- 选择目标存储后端（根据软件类型自动筛选）
- 上传制品文件（支持大文件分片上传）
- 自动提取元数据（大小、格式、校验和）
- 上传进度显示

**附件管理**：
- 上传配套文档（使用手册、部署指南等）
- 上传安全报告（漏洞扫描报告、合规证明等）
- 附件列表管理（添加、删除、预览）
- **附件限制**：单文件最大 100MB
- **附件存储**：本地磁盘存储（非 NAS）

**软件版本状态机**（REQ-320）：

```
                    提交审核
    草稿 ───────────────────────▶ 待审核
     ▲                              │
     │         保存草稿             │ 审批通过
     └──────────────────────────────┤
                                    ▼
                              ┌───────────┐
                              │   已发布  │◀────────────────┐
                              └───────────┘                 │
                                    │                       │
                                    │ 下架                  │ 重新上架
                                    ▼                       │
                              ┌───────────┐   归档          │
                              │   已下架  │────────────────▶│ 已归档
                              └───────────┘                 │
                                                            │
                                    驳回 ───────────────────┘
                                    (从待审核)
```

**状态说明**：
| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 草稿 | 编辑中，未提交审核 | 编辑、删除、提交审核 |
| 待审核 | 等待管理员审批 | 取消申请（退回草稿） |
| 已发布 | 已通过审核，可在门户订购 | 下架、编辑描述信息 |
| 已下架 | 不再提供新订购，已有订购仍可使用 | 重新上架、归档 |
| 已归档 | 完全不可见，仅后台可查 | 无 |

#### 4.3.2 软件审核与发布

**需求编号**: REQ-360 ~ REQ-370

**发布申请**：
- 软件管理员提交发布申请
- 系统管理员审批
- 审批操作：通过 / 驳回（填写原因）
- 审批历史记录

#### 4.3.3 版本管理

**需求编号**: REQ-330 ~ REQ-340

**版本策略**：
- 一个软件支持多个版本共存
- 新版本发布时，旧版本保持可订购状态
- 各版本独立管理（独立发布、独立订购）
- 版本可单独下架（不影响其他版本）

**版本维护**：
- 上传新版本
- 编辑版本信息
- 版本下架/重新上架

**版本升级策略**（REQ-330）：
- 新版本发布后，自动通知已订购旧版本的用户（站内信+邮件）
- 提供"一键升级订购"功能，保留原有业务系统关联
- 旧版本订购保留有效期，标记为"有新版本可用"
- 支持设置"推荐版本"，门户默认展示推荐版本

**需求编号**: REQ-350

- 编辑软件基础信息（描述、标签等）
- 软件整体下架（所有版本不可新订购，已有订购不影响）
- 软件归档（完全不可见）

### 4.5 订购与下载（开发人员）

#### 4.5.1 订购申请

**订购表单**：
- **选择版本**（必选）：从该软件的所有已发布版本中选择特定版本
- **使用业务系统**（多选，至少一个）：
  - 从系统预置的业务系统列表中选择（可多选）
  - 或点击"新建业务系统"（新建审批通过后可添加）
- 使用场景描述
- 预计使用规模（如：服务器数量、实例数量）

**版本选择说明**：
- 订购与特定版本绑定
- 不同版本的订购相互独立
- 升级版本需重新订购

**新建业务系统流程**：
- 开发人员填写业务系统信息
- 提交新建申请
- 系统管理员审批
- 审批通过后自动关联到当前订购

**订购状态**：
- 待审批（等待审批）
- 已通过（可使用）
- 已驳回（查看原因，可重新申请）
- 已过期（令牌过期，需重新申请）

#### 4.5.2 审批管理

**审批人**：软件管理员和系统管理员均可审批

**审批操作**：
- 通过：生成下载令牌
- 驳回：填写驳回原因
- 可设置审批有效期（默认长期有效）

**审批提醒**：审批人收到待审批通知（邮件/站内信）

#### 4.5.3 下载获取

**审批通过后提供**：

支持多种获取方式（根据软件类型自动适配）：

| 软件类型 | 存储后端 | 获取方式 |
|---------|---------|---------|
| Docker 镜像 | Harbor | docker pull 命令 |
| Helm Chart | Harbor | helm pull 命令 |
| Maven | Nexus | Maven 依赖坐标（XML/Gradle） |
| NPM | Nexus | npm install 命令 |
| PyPI | Nexus | pip install 命令 |
| 通用文件 | NAS | HTTPS 直接下载链接 |

**令牌机制（全局统一配置）**：
- 系统管理员在后台配置默认策略：
  - 默认有效期：1小时 / 1天 / 7天 / 30天 / 永久
  - 默认最大下载次数：不限 / 1次 / 5次 / 10次 / 自定义
- 生成唯一下载令牌
- 令牌状态：有效、已过期、已用完、已撤销
- 支持提前撤销令牌
- **令牌续期**（REQ-540）：
  - 有效期内支持续期，每次续期延长原有效期
  - 最大续期次数：3次
  - 使用期间活跃（下载操作）可自动延长有效期

**令牌状态流转**：
```
有效 ──使用/续期──▶ 有效（延长有效期）
  │
  ├─过期时间到达──▶ 已过期
  ├─下载次数用尽──▶ 已用完
  └─管理员撤销───▶ 已撤销
```

**下载页面展示**：
- 复制按钮（一键复制命令/链接）
- 二维码（手机扫码获取）
- 有效期倒计时
- 剩余次数进度条
- 使用说明文档

### 4.9 许可证合规分析

```json
    "ranges": [{"introduced": "4.0.0", "fixed": "4.17.21"}]
  }]
}
```

- 按严重程度分类统计

#### 4.10.3 漏洞告警
- **新软件录入时**：自动扫描并提示漏洞
- **CVE 数据更新时**：自动扫描存量软件
- **告警通知**：受影响软件的业务系统负责人收到通知
- **修复版本推荐**：提示升级到哪个版本可修复漏洞

#### 4.10.4 漏洞跟踪
- 漏洞修复状态跟踪（未修复/已修复/已评估接受风险）
- 修复计划制定
- 修复验证

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 日活用户支持 | 1000+ | 同时在线 |
| 峰值并发 | 100 | 同时操作 |
| 文件上传 | 10GB+ | 支持断点续传，速度≥10MB/s |
| 页面响应 | < 2s | 普通操作（95%请求） |
| 复杂查询 | < 5s | 大数据量统计查询 |
| 搜索响应 | < 500ms | 关键字搜索（95%请求） |
| 数据库性能 | 单表1000万条 | 查询性能不下降 |

### 4.2 安全要求
- 用户认证：JWT Token + Session
- 传输加密：HTTPS
- 敏感信息加密存储（密码、Token）
- 操作日志记录
- 权限控制：RBAC 模型

### 4.3 可用性要求
- 系统可用性：99.9%
- 数据备份：每日自动备份
- 故障恢复：支持快速恢复

### 4.4 扩展性要求
- 存储后端可插拔（后续可扩展其他类型）
- 扫描平台可对接（预留接口）
- 通知渠道可扩展

### 4.5 数据保留策略（REQ-950）

| 数据类型 | 保留期限 | 说明 |
|----------|----------|------|
| 已删除软件元数据 | 1年 | 仅管理员可见，用于审计 |
| 登录日志 | 1年 | 安全审计 |
| 操作日志 | 3年 | 完整操作追溯 |
| 下载日志 | 2年 | 使用情况分析 |
| 令牌记录 | 30天（过期后） | 下载追溯 |
| 系统日志 | 6个月 | 故障排查 |

**数据清理**：
- 自动清理：超过保留期的数据自动归档或删除
- 归档存储：清理前的数据导出到归档存储
- 手动清理：管理员可手动触发历史数据清理

### 4.6 审计要求（REQ-960）

**审计日志内容**：
- 所有数据修改操作（谁、何时、做了什么、结果）
- 敏感操作：删除、权限变更、配置修改
- 登录登出记录
- 令牌生成与使用

**审计功能**：
- 操作追溯：支持按用户、时间、资源类型查询
- 敏感操作双重确认：删除、权限变更需二次确认
- 审计日志导出：支持 CSV/PDF 格式导出
- 防篡改：审计日志写入独立存储，不可修改
- 告警机制：敏感操作实时通知安全管理员

### 4.7 缓存策略（REQ-970）

**Redis 用途**：
| 用途 | 说明 | 过期策略 |
|------|------|----------|
| 会话/Token | JWT 黑名单、Refresh Token | Token 有效期一致 |
| 热点数据 | 软件列表、用户信息 | TTL 5分钟 |
| 用户权限 | 角色权限缓存 | TTL 10分钟 |
| 统计数据 | 门户统计、用量统计 | TTL 1小时 |
| 限流计数 | API 限流、登录频率 | 按限流窗口 |
| 分布式锁 | 并发控制 | 业务完成后释放 |

## 5. 技术选型建议

### 5.1 后端
- **框架**：Spring Boot 3.4 + Java 21
- **数据库**：PostgreSQL 15（主库）+ Redis（缓存/会话）
- **ORM**：Spring Data JPA
- **迁移**：Flyway

#### SBOM 与依赖解析
- **Maven 依赖解析**：使用 Maven Invoker API 或解析 pom.xml
- **NPM 依赖解析**：使用 package.json 和 package-lock.json 解析
- **SBOM 生成**：使用 CycloneDX Java 库
- **SPDX 支持**：使用 Spdx-Java-Library

#### 许可证识别
- **License 文本匹配**：使用 SPDX License List
- **License 文件识别**：扫描 LICENSE/NOTICE/COPYING 文件
- **SNIPPET 匹配**：基于哈希的许可证文本匹配

#### 漏洞管理（离线）
- **CVE 数据存储**：本地数据库存储 NVD/OSV 数据
- **版本范围匹配**：使用 Semver4j 进行语义化版本匹配
- **漏洞扫描引擎**：本地扫描，不依赖外部 API

### 5.2 前端
- **框架**：Vue 3.4 + TypeScript 5.0
- **构建工具**：Vite 5.0
- **组件库**：Element Plus
- **状态管理**：Pinia 2.0
- **路由**：Vue Router 4.0
- **HTTP 客户端**：Axios
- **样式**：SCSS + CSS 变量

**选型理由**：Vue 3 在国内企业级应用中有更好的生态支持和开发者熟练度，与 Spring Boot 后端配合成熟稳定。

#### 图表与可视化（运营看板）
- **图表库**：ECharts 5.0
- **雷达图**：技术雷达展示
- **关系图谱**：依赖关系可视化（ECharts Graph 或 G6）
- **数据表格**：Element Plus Table + vxe-table（大数据量）

#### 文档编辑
- **富文本编辑**：Tiptap 或 WangEditor（知识库文档）
- **Markdown 支持**：marked.js + highlight.js（SBOM/报告展示）

### 5.3 部署
- **容器化**：Docker
- **编排**：Docker Compose（开发）/ Kubernetes（生产）
- **反向代理**：Nginx

## 6. 核心业务流程图

### 6.1 存储后端纳管流程
```
系统管理员
    ↓
进入【存储管理】
    ↓
选择类型（Harbor/Nexus/NAS）
    ↓
填写配置信息（地址、认证等）
    ↓
点击【连接测试】
    ↓
测试成功 → 保存配置 → 纳管成功
测试失败 → 提示错误 → 修改重试
```

### 6.2 开源软件发布流程
```
软件管理员
    ↓
进入【软件管理】→ 点击【录入软件】
    ↓
填写基本信息（名称、版本、类型等）
    ↓
上传制品文件 → 选择目标存储后端
    ↓
上传附件（文档、安全报告）
    ↓
保存草稿 / 提交发布申请
    ↓
系统管理员审批
    ↓
审批通过 → 软件上架门户
审批驳回 → 查看原因 → 修改重提
```

### 6.3 订购使用流程
```
开发人员
    ↓
浏览门户 → 选择软件 → 点击【订购】
    ↓
填写订购信息：
  - 选择业务系统（从列表选或新建）
  - 如新建业务系统 → 提交新建申请 → 等待审批
    ↓
提交订购申请
    ↓
软件管理员/系统管理员审批
    ↓
审批通过 → 获取下载令牌
    ↓
查看下载命令/链接（带有效期）
    ↓
使用 docker pull / npm install / 下载
    ↓
系统记录使用情况
```
